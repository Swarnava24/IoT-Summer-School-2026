# I2C Protocol in IoT Sensor Communication

## What is I2C?
**I2C (Inter-Integrated Circuit)** is a serial communication protocol that lets a microcontroller (like an Arduino) talk to multiple sensors/devices using only **two wires**, instead of dedicating separate pins to each device. This makes it ideal for IoT systems where several sensors need to share the same microcontroller with limited pins.

It works on a **master-slave** model: the microcontroller (master) initiates communication, and each sensor (slave) responds only when its unique address is called.

## SDA and SCL lines
I2C uses exactly two lines, shared by all connected devices:

- **SDA (Serial Data line):** Carries the actual data being sent between master and slave, in both directions.
- **SCL (Serial Clock line):** Carries the clock signal generated by the master, which synchronizes when data bits are read/written on SDA.

Both lines are "open-drain" and need pull-up resistors (often built into sensor breakout boards) to sit at HIGH when idle.

## How I2C addressing works
Every I2C device on the bus has a unique **7-bit (sometimes 10-bit) address**. When the master wants to communicate with a specific sensor, it first sends that sensor's address on the bus. All connected devices "hear" the address, but only the device whose address matches responds — every other device stays silent. This is what allows many sensors to share just the SDA/SCL lines without interfering with each other.

On Arduino UNO, the I2C pins are fixed: **A4 (SDA)** and **A5 (SCL)**.

## Three common IoT sensors that use I2C
1. **BMP280 / BME280** – atmospheric pressure, temperature, and humidity sensor
2. **MPU6050** – accelerometer + gyroscope (motion sensing)
3. **16x2 I2C LCD display module** (with PCF8574 backpack) – for showing sensor data locally
